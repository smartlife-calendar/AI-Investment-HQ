## 報告輸出格式

每個框架輸出必須包含 **5 個區段**：

```
一、核心計算
   → 所有公式顯示計算過程和數字

二、指標評分
   → 表格：指標 | 數值 | 標準 | ✅/❌/⚠️

三、市場情緒評估
   → 結合 VIX/貪婪指數/板塊ETF 判斷進場時機

四、主要風險
   → 前3項風險，每項附具體數字

五、估值結論
   → 悲觀目標價 / 基準目標價 / 樂觀目標價
   → 評級：強力買進/買進/觀望/賣出/強力迴避
   → 升級觸發條件 / 降級觸發條件
```

## 數值品質保證 (Data Quality Gates)

| 檢查項目 | 機制 |
|---|---|
| Ticker 有效性 | Yahoo Finance 確認後才進入分析 |
| 目標價合理性 | 非價值框架：目標價需在當前股價 15%-800% 之間 |
| 目標價排序 | 自動確保 悲觀 ≤ 基準 ≤ 樂觀 |
| 評級準確性 | 只在「估值結論」區段搜尋評級，避免觸發條件干擾 |
| YoY 數據配對 | get_all_annual() 確保同一時期數據比較 |
| 新上市公司 | 無 10-K 時自動使用最新 10-Q 季報 |
| GrossProfit 缺失 | 自動計算 Revenue - COGS |
| 台股 Equity | 使用 FinMind 正確 key（`Equity`） |
